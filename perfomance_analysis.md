# Анализ производительности
## 8 Управление проектами

что имеется (API): 
- управление пользователями (регистрация, поиск, аутентификация JWT)
- управление проектами (создание, просмотр)
- управление задачами (создание в проекте)
- комментирование задач (с вложенными ответами через MongoDB)
- две СУБД: PostgreSQL (пользователи, проекты, задачи) + MongoDB (комментарии - c API; история, уведомления - без)

### Hot Paths: 
1. Аутентификация/логин
2. Поиск пользователя по маске имени
3. Получение всех проектов
4. Создание задачи
5. Получение комментариев задачи

### Медленные операции (с высокими задержками)
Обращения к PostgreSQL:
1. Поиск по маске имени ~5.6 мс (с индексом) + растет с размером таблицы
2. Получение всех проектов ~9.3 мс (под нагрузкой 1079 RPS)
3. JOIN-запросы в задачах ~15-30 мс

Обращения к MongoDB:
1. Получение комментариев с replies (нужно загрузить весь комментарий с массивом replies, сетевой вызов) ~11.5 мс (под нагрузкой 876 RPS)
2. Вставка комментария (Запись + обновление индексов) ~2-5 мс
3. Добавление reply ($push в массив replies (атомарная операция)) ~3-8 мс

Внешние зависимости:
|Компонент|Проблема|Задержка|
|---|---|---|
|PostgreSQL|Сетевой RTT между контейнерами|~0.2-0.5 мс|
|MongoDB|Сетевой RTT + сложность запроса|~0.5-2 мс|
|JWT верификация|Подпись HMAC-SHA256|~0.05-0.1 мс|

## Узкие места для оптимизации
Критические:  
PostgreSQL LIKE поиск (713 RPS при 5.6 мс)  
Отсутствие кеширования (каждый запрос идет в БД)  
Нет защиты от брутфорса (716 RPS для логина без ограничений)  

Средние:  
Сериализация JSON  
MongoDB embeds  

### Метрики для мониторинга
Cache Hit Ratio  
Rate Limit Reject Rate  
PostgreSQL Query Time (p95)  
MongoDB Query Time (p95)  
Login Failure Rate per IP

|Метрика|Текущее значение|Целевое значение|
|---|---|---|
|Cache Hit Ratio|0% (нет кеша)|>80%|
|Rate Limit Reject Rate|0% (нет лимитов)|<1%|
|PostgreSQL Query Time (p95)|~58.6 мс (max)|<30 мс|
|MongoDB Query Time (p95)|~81.5 мс (max)|<20 мс|
|Login Failure Rate per IP|не ограничен|≤5 попыток/мин|

## Ожидаемое улучшение после оптимизации
|Endpoints|Без оптимизации|С кешированием|Ускорение|
|---|---|---|---|
|GET /api/projects|1079 RPS / 9.3ms|ожидается 8000+ RPS / 2-3ms|~7x|
|GET /api/comments|876 RPS / 11.5ms|ожидается 5000+ RPS / 3-4ms|~5x|
|POST /api/login|716 RPS (неограниченно)|5 попыток/мин на IP|защита от брутфорса|
 
Cache hit rate ожидается: >90% для проектов (TTL 5 минут), >80% для комментариев (TTL 1 минута). инвалидация кеша происходит при создании новых проектов и добавлении комментариев.
